## スライド4: 移行対象ユーザー一覧CSV

### 必須項目

```csv
UserPrincipalName,SamAccountName,PrimarySmtpAddress,Aliases,備考
user1@contoso.local,user1,user1@example.co.jp,alias1@example.co.jp;sales@example.co.jp,営業部
user2@contoso.local,user2,user2@example.co.jp,,
```

**記入ルール**:
- 1行1ユーザー
- エイリアスはセミコロン(`;`)区切り
- プライマリアドレスは`PrimarySmtpAddress`に記載

**確認事項**: 同一人物が複数ADアカウントを持つケースはありますか？

## スライド5: 2. 現行システム設定

### 🔍 設定情報の提供が必要（優先度：高）

| システム | 確認内容 | 提供方法 |
|---|---|---|
| **FireEye** | 40ドメインの転送先設定（→AWS DMZ SMTP確認） | スクリーンショット or 設定エクスポート |
| **GuardianWall** | 現行ルール・ポリシー（GWC移行用） | 設定エクスポート（可能なら） |
| **AWS DMZ SMTP** | transport設定（ドメインごとの転送先） | 設定ファイル or 確認方法 |

**目的**: 移行後も整合性を保つため

## スライド6: 3. ID運用の現状

### 👤 ID作成・管理フローの確認（優先度：高）

**現在のフロー**:
```
ユーザー申請 → 電話帳システム → 統合アカウントシステム → AD
```

**確認したいこと**:

| フェーズ | 確認事項 |
|---|---|
| **Joiner（作成）** | どういう時に作られるか？<br>命名規則は？<br>承認フローは？ |
| **Mover（変更）** | 部署異動時のアドレス変更ルールは？<br>エイリアス追加のルールは？ |
| **Leaver（削除）** | 退職時のメール対応ルールは？<br>データ保持期間は？ |

**目的**: EXO移行後のID運用方針を策定するため

## スライド7: ID運用の詳細確認

### 具体的な質問

**作成（Joiner）**

- ❓ ID作成の申請者は？（ユーザー本人？管理者？）
- ❓ 承認者は？
- ❓ 複数アドレスを付与する基準は？

**変更・削除（Mover/Leaver）**
- ❓ 部署異動時、メールアドレスは変更しますか？
- ❓ 退職時のメールボックスはどうしますか？（削除？転送？保持期間？）

## スライド8: 4. 技術的確認事項

### ⚙️ システム連携の有無確認（優先度：中〜高）

| # | 確認事項 | 理由 | 回答例 |
|---|---|---|---|
| 1 | **EWSを使用しているシステムはあるか** | EXO環境でEWSがONになっている理由確認 | なし / ○○システム |
| 2 | **内部DMZ SMTPを使っている他システムはあるか** | EXOからの接続許可時の影響確認 | なし / AD通知等 |
| 3 | **共有アドレスの件数と用途** | Shared Mailbox / DG / M365 Group への振り分け | ○件 / ○○用途 |
| 4 | **システムメールの送信元・経路** | 移行後の経路確認 | 統合IDシステム等 |

**影響範囲の把握のため、ご回答をお願いします**
